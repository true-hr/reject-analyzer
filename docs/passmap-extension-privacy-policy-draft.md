# PASSMAP 선택 텍스트 보내기 확장 개인정보처리방침 (초안)

> **이 문서는 초안입니다.**
> 법적 효력을 가지는 최종 개인정보처리방침이 아니며, 실제 배포 전 법무 / 운영 측 검토를 거쳐야 합니다.
> 본 문서는 사용자의 검토를 돕기 위한 정리본일 뿐, 본 문서만으로 법적 고지를 갈음하지 않습니다.

작성일: 2026-05-23 (초안)
대상 확장: PASSMAP 선택 텍스트 보내기 (`extension/passmap-selection-import`)
대상 도메인: `https://reject-analyzer.vercel.app/`

---

## 1. 개요

PASSMAP 선택 텍스트 보내기 확장(이하 "본 확장")은 사용자가 ChatGPT, Gemini, Claude 또는 임의의 웹페이지에서 **직접 드래그하여 선택한 텍스트**를 PASSMAP의 경험 기록 입력창으로 한 번에 전달하기 위한 보조 도구입니다.

본 확장은 사용자가 선택한 텍스트 외 어떠한 데이터도 자동으로 수집하지 않습니다.

본 개인정보처리방침은 본 확장 자체의 동작에 한정됩니다. PASSMAP 웹 서비스 본체(`https://reject-analyzer.vercel.app/`)에 저장되는 데이터에 대한 처리 방침은 PASSMAP 본체의 별도 약관 및 개인정보처리방침에 따릅니다.

---

## 2. 처리하는 정보

본 확장이 처리하는 정보는 다음 한 가지뿐입니다.

### 사용자가 직접 드래그하여 선택한 텍스트

- 처리 시점: 사용자가 우클릭 컨텍스트 메뉴 `패스맵에서 경험 찾기` 항목을 클릭한 시점에 한정
- 처리 범위: 최소 30자 이상, 최대 50,000자까지. 30자 미만 선택은 무시되고, 50,000자 초과 시 앞에서 50,000자까지만 처리
- 처리 경로:
  1. 브라우저 표준 `chrome.contextMenus` API가 전달하는 `info.selectionText` 값을 받음
  2. `chrome.storage.local` 의 `PASSMAP_EXTERNAL_INTAKE_BRIDGE` 키에 일회용으로 저장
  3. PASSMAP 도메인 새 탭에서 동작하는 content script가 해당 값을 읽어 그 페이지의 `sessionStorage` 의 `PASSMAP_EXTERNAL_INTAKE` 키에 옮긴 뒤, `chrome.storage.local` 의 bridge 값은 즉시 삭제
- 처리 목적: PASSMAP 웹앱이 경험 후보 추출 입력창에 자동으로 값을 채워 사용자가 검토할 수 있도록 하기 위함

### 보내지는 데이터의 구조

```json
{
  "version": 1,
  "sourceMode": "ai_conversation",
  "importMethod": "browser_extension_selection",
  "rawText": "사용자가 선택한 텍스트",
  "savedAt": 1779450000000
}
```

`rawText` 외의 필드는 메타데이터입니다. 별도의 식별자, IP, 디바이스 정보, 위치 정보는 포함되지 않습니다.

---

## 3. 처리하지 않는 정보

본 확장은 아래 항목들을 **읽지도, 저장하지도, 전송하지도 않습니다.**

- ChatGPT / Gemini / Claude 및 기타 AI 서비스의 전체 대화 내역
- 사용자가 선택하지 않은 페이지 본문, 사이드바, 시스템 프롬프트, 첨부 파일
- 쿠키, 로그인 세션, 인증 토큰
- 비밀번호, 자동완성 데이터, 결제 정보
- 브라우징 기록, 방문 사이트 목록, 즐겨찾기
- 마우스 움직임, 키 입력 이벤트, 화면 캡처
- IP 주소, 디바이스 식별자, 위치 정보
- 광고 식별자, 추적 픽셀, 외부 분석 SDK

본 확장은 ChatGPT / Gemini / Claude 등 외부 AI 서비스 도메인에 대해 `host_permissions` 를 요청하지 않으며, 해당 페이지의 DOM에 접근하지 않습니다. 브라우저의 표준 컨텍스트 메뉴 API가 사용자가 "선택한 텍스트" 만 전달해 주기 때문에 별도의 페이지 접근이 필요하지 않습니다.

---

## 4. 데이터 전송 범위

본 확장 자체는 **외부 네트워크 호출을 수행하지 않습니다.**

모든 데이터 이동은 사용자의 브라우저 내부에서 일어나며, 구체적으로는 다음 경로로만 이동합니다.

1. 사용자의 선택 텍스트 → 브라우저의 `chrome.storage.local` (확장 전용 영역)
2. `chrome.storage.local` → 사용자의 새 탭에서 열린 PASSMAP 도메인의 `sessionStorage`
3. 그 시점에 `chrome.storage.local` 의 bridge 값은 삭제됨

PASSMAP 도메인의 `sessionStorage` 에 들어간 텍스트는 PASSMAP 웹앱이 사용자에게 표시하기 위한 용도이며, **사용자가 PASSMAP 화면에서 명시적으로 저장/분석 버튼을 누르기 전까지는** 패스맵 서버로 전송되지 않습니다.

본 확장은 본 확장의 자체 서버를 운영하지 않습니다. 본 확장 개발자가 별도로 운영하는 분석/로깅 서버는 존재하지 않습니다.

---

## 5. 보관 및 삭제

| 항목 | 보관 위치 | 보관 기간 |
|---|---|---|
| 선택 텍스트(bridge payload) | `chrome.storage.local` | 새 탭의 content script가 한 번 읽는 즉시 삭제. 길어도 약 수 초 |
| 선택 텍스트(sessionStorage) | PASSMAP 도메인의 sessionStorage | 해당 탭이 열려 있는 동안만. 탭을 닫으면 자동 삭제 |
| 패스맵 서버에 저장된 경험 후보 | 패스맵 백엔드 | PASSMAP 본체 정책에 따름 (본 확장과 무관) |

bridge payload는 비정상 종료나 도메인 라우팅 실패 등의 예외 상황에서도 5분이 지나면 stale 처리되어 무시되며, 다음 정상 흐름이 발생하는 시점에 정리됩니다.

사용자가 확장을 제거하면 `chrome.storage.local` 의 모든 데이터는 브라우저에 의해 자동 삭제됩니다.

## 6. 권한 사용 사유

본 확장이 `manifest.json` 에서 요청하는 권한과 그 이유입니다.

| 권한 | 사용 이유 |
|---|---|
| `contextMenus` | 우클릭 메뉴 `패스맵에서 경험 찾기` 항목을 추가하기 위해 |
| `storage` | 선택 텍스트를 새 탭으로 안전하게 넘기는 일회용 브라우저 내부 저장소(`chrome.storage.local`) 로 쓰기 위해. 외부 서버 전송이 아님 |
| `tabs` | PASSMAP 도메인을 새 탭으로 열기 위해. 다른 탭의 URL이나 내용은 읽지 않음 |
| `host_permissions: https://reject-analyzer.vercel.app/*` | PASSMAP 도메인에서만 content script가 동작하도록 제한하기 위해 |

`<all_urls>` 또는 와일드카드 host_permissions는 사용하지 않습니다.

본 확장은 외부 분석 SDK, 광고 SDK, 추적 픽셀을 포함하지 않습니다.

## 8. 민감 정보 처리 안내

본 확장은 사용자가 직접 선택한 텍스트만 처리하므로, 어떤 내용을 보낼지는 전적으로 사용자가 결정합니다.

다음 내용은 드래그 선택 전 단계에서 사용자가 직접 제외해 주시기 바랍니다.

- 주민등록번호, 여권번호, 운전면허번호 등 고유식별정보
- 신용카드 번호, 계좌번호, 결제 정보
- 비밀번호, OTP, API 키, 토큰
- 의료 정보, 종교, 사상, 정치적 견해, 노조 가입 여부 등 민감정보
- 미공개 인사 정보, 미공개 사업 기밀, 미공개 법무 정보
- 미성년자에 관한 정보

본 확장은 텍스트 내용을 사전에 필터링하지 않습니다. 사용자가 보낸 내용은 PASSMAP 웹앱의 입력창에 그대로 표시되며, 사용자가 PASSMAP 본체에서 명시적으로 저장 버튼을 누르면 PASSMAP 본체의 정책에 따라 처리됩니다.

## 10. 안전 조치

- 본 확장은 코드 전체가 패키지 내부에 정적으로 포함되며, 원격 코드 로딩(remote code) 을 사용하지 않습니다.
- 본 확장이 처리하는 데이터는 사용자 브라우저 내부에서만 이동하며, 본 확장의 자체 외부 서버는 존재하지 않습니다.
- `host_permissions` 는 PASSMAP 운영 도메인 한 곳으로 제한합니다.
- bridge payload는 단일 사용 후 즉시 삭제하며, 5분 stale 정책으로 잔류를 방지합니다.
