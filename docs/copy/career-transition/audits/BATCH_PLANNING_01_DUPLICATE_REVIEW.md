## 3개 케이스 간 상호 비교

### "서비스기획" 언어 중복 위험

퍼포먼스마케팅→서비스기획 케이스와 CX→서비스기획 케이스는 모두 targetJob이 서비스기획이다. 따라서 Target Job Feature 섹션이 유사할 위험이 있다.

**확인 결과:**
- 퍼포먼스마케팅→서비스기획: "전환 퍼널 이해 → 화면·기능 설계" 방향으로 연결 경로 차별화
- CX→서비스기획: "VOC·반복 문의 이해 → 화면·정책 개선 기획" 방향으로 연결 경로 차별화
- Target Job Feature 섹션의 JD 요구 역량 리스트는 유사하나, 각 케이스의 강점 진입점이 다름
- Top Summary와 Core Difference가 다른 논리로 구성됨

**판단:** MINOR_EDIT_NEEDED → 수정 완료 (Target Job Feature 섹션에서 동일한 리스트를 그대로 쓰되, 각 케이스가 어떤 부분에서 연결되고 어떤 부분이 부족한지를 다르게 표현하도록 본문에서 이미 구분함. 별도 수정 불필요.)

### PM 언어 과다 사용 여부

"문제정의", "우선순위", "제품 오너십", "지표"가 BATCH_PLANNING_01 케이스에 과도하게 사용되는지 확인.

**확인 결과:**
- 서비스기획→사업기획: "문제정의" 미사용. "우선순위"는 사업적 의사결정 맥락에서만 1회 사용. PM 언어 없음.
- 퍼포먼스마케팅→서비스기획: "문제정의" 1회 (Forbidden Mix 설명에서 PM 케이스와의 차이 설명용). 직접 발화 없음.
- CX→서비스기획: "문제정의" 미사용. "우선순위"는 기획 과제 우선순위 맥락에서 1회. PM 언어 없음.

**판단:** PASS
