# 산출물 D 상세 정의: 용어와 톤 정리

## D-0. 이 산출물의 목적

`HomeDashboard` 전체 문구를 `업무 관리 SaaS` 톤에서 `경험 정리 / 커리어 분석` 톤으로 전환한다.

사용자가 화면을 보면서 아래처럼 느끼는 것이 목표다.

> 내 업무 기록이 단순 저장되는 것이 아니라, 취업/이직에 쓸 수 있는 경험 신호와 이력서 후보로 정리되고 있구나.

이 산출물은 모든 영역의 카피 기준이다. 산출물 A/B/C를 구현할 때도 이 파일의 톤 규칙을 따른다.

---

## D-1. 현재 대체해야 할 제품 인상

현재 화면에서 피해야 할 인상은 아래다.

- 업무 관리 도구
- 생산성 앱
- 단순 기록 저장소
- 리포트 생성기
- 이력서 문장 생성기
- 관리자용 분석 화면

이번 작업에서 지향하는 인상은 아래다.

- 경험 정리 대시보드
- 기록 기반 커리어 분석 플랫폼
- 직무 연결 가능성을 보여주는 화면
- 이력서 후보를 발굴하는 도구
- 다음 행동을 알려주는 취업/이직 준비 도구

---

## D-2. 핵심 용어 치환표

아래 용어 치환을 기본값으로 사용한다.

| 기존 표현 | 권장 표현 | 비고 |
|---|---|---|
| 업무 관리 | 경험 정리 | 메뉴명 전체 변경은 신중하게, 내부 문구 우선 |
| 업무 기록 | 경험 기록 | 입력 맥락에서는 자연스러운 경우 `기록`만 써도 됨 |
| 최근 리포트 | 최근 커리어 인사이트 | 메인 섹션명에서는 `리포트` 지양 |
| 이력서 문장 | 이력서 후보 | 자동 생성 서비스처럼 보이지 않게 |
| 이력서 문단 | 이력서 후보 | `문단`보다 `후보` 우선 |
| 만든 이력서 문장 | 정리된 이력서 후보 | 성과물보다 후보/재료 느낌 |
| 쌓이고 있는 커리어 방향 | 연결 가능성이 보이는 직무 | 추상 표현 제거 |
| 추천 액션 | 다음 추천 행동 | 행동 이유와 함께 제시 |
| 약한 항목 | 보완하면 좋은 신호 | 평가 느낌 완화 |
| 기록 없음 | 첫 기록 유도 문구 | 단독 노출 지양 |
| 0건 | 첫 기록 대기 중 / 아직 쌓는 중 | 숫자 부족감 지양 |

## D-3. 기본 문장 톤

### 권장 톤

- 부드럽지만 지나치게 가볍지 않은 톤
- 가능성형 표현
- 사용자가 평가받는 느낌보다 안내받는 느낌
- 기록의 가치를 직무/이력서 후보와 연결하는 톤
- `AI가 단정한다`보다 `기록에서 보이는 신호를 정리한다`는 톤

### 권장 표현

```text
경험 신호가 보입니다.
반복적으로 나타나고 있어요.
연결 가능성이 있습니다.
이력서 후보로 정리해볼 수 있어요.
기록을 더 남기면 분석 정확도가 높아져요.
보완하면 설득력이 높아져요.
아직 덜 드러난 신호를 더 선명하게 만들 수 있어요.
```

### 피해야 할 표현

```text
확실합니다.
검증되었습니다.
이 직무가 맞습니다.
합격 가능성이 높습니다.
부족합니다.
약합니다.
실패했습니다.
0건입니다.
기록이 없습니다.
이력서 문장을 만들었습니다.
```

## D-4. 상태별 카피 기준

### 기록이 없는 사용자

목표:
- 부족함을 지적하지 않는다.
- 첫 기록을 남기면 무엇이 생기는지 보여준다.

권장 카피:

```text
아직 기록이 없어도 괜찮아요. 오늘 한 일을 한 줄만 남기면 AI가 경험 신호를 찾아드릴게요.
```

```text
첫 기록을 남기면 경험 흐름과 연결 가능 직무를 함께 정리해드릴게요.
```

피해야 할 카피:

```text
기록 없음
아직 기록이 없습니다
0건
표시할 데이터가 없습니다
```

### 기록이 적은 사용자

목표:
- 초기 신호를 보여주되, 더 쌓으면 좋아진다는 기대를 준다.

권장 카피:

```text
아직 기록은 많지 않지만, 남겨둔 경험에서 보이는 초기 신호를 정리했어요.
```

```text
기록을 몇 개 더 남기면 반복 경험과 연결 직무를 더 정확하게 볼 수 있어요.
```

### 기록이 충분한 사용자

목표:
- 기록 기반 분석이 진행 중이라는 신뢰감을 준다.

권장 카피:

```text
최근 기록에서 반복적으로 나타나는 경험 신호와 직무 연결 가능성을 정리했어요.
```

```text
이 신호는 이력서 후보로 정리해볼 수 있어요.
```

## D-5. 섹션별 카피 기준

### 상단 카드

- 핵심어: `최근 경험 분석 결과`, `반복 경험 신호`, `연결 가능성이 보이는 직무`, `이력서 후보`
- 피해야 할 핵심어: `이번 주 만든 이력서 문장`, `기록 상태`, `0건`

### 캘린더

- 핵심어: `경험 흐름`, `이 날짜의 경험 신호`, `다음 행동`, `보완하면 좋은 정보`
- 피해야 할 핵심어: `기록 없음`, `표시할 강점 없음`, `0건`

### 하단 인사이트

- 핵심어: `최근 커리어 인사이트`, `최근 경험 흐름`, `현재 강점 및 부족 신호`, `다음 추천 행동`
- 피해야 할 핵심어: `최근 리포트`, `추천 액션`, `약한 항목`

## D-6. 확정형/과장형 표현 금지

PASSMAP은 사용자의 직무 적합성이나 합격 가능성을 보장하면 안 된다.

피해야 할 표현:

```text
이 직무가 맞습니다.
합격 가능성이 높습니다.
기업이 좋아합니다.
검증된 강점입니다.
이 경험은 반드시 이력서에 들어가야 합니다.
```

권장 표현:

```text
이 직무와 연결 가능성이 보입니다.
이력서 후보로 정리해볼 수 있어요.
이 경험은 직무 이해도와 실행력을 보여주는 재료가 될 수 있어요.
기록을 더 남기면 판단 근거가 더 선명해져요.
```

## D-7. `이력서 후보` 표현 규칙

PASSMAP은 단순히 이력서 문장을 만들어주는 서비스처럼 보이면 안 된다.

따라서 메인 CTA/카피에서는 아래를 우선한다.

```text
이력서 후보 보기
이력서 후보로 정리하기
이력서 초안 후보 확인하기
```

아래 표현은 되도록 피한다.

```text
이력서 문장 만들기
이력서 자동 생성
이력서로 변환하기
```

단, 기존 기능명이 이미 `이력서 보기`인 경우에는 기능 연결을 위해 유지할 수 있다. 이 경우 주변 설명에서 `후보`와 `정리` 표현을 보완한다.

---

## D-8. 부족 신호 표현 규칙

부족 신호는 사용자의 결함처럼 보이면 안 된다.

피해야 할 표현:

```text
부족합니다
약합니다
미흡합니다
문제입니다
```

권장 표현:

```text
아직 덜 드러납니다
더 선명하게 만들 수 있어요
보완하면 설득력이 높아져요
다음 기록에서 함께 남기면 좋아요
```

예시:

```text
성과 수치는 아직 덜 드러납니다. 처리 건수, 개선률, 사용자 반응을 함께 남기면 이력서 후보의 설득력이 높아져요.
```

## D-9. CTA 카피 기준

CTA는 단순 명령형보다 기대 효과가 보이는 문구를 우선한다.

권장 CTA:

```text
기록 이어가기
첫 경험 기록하기
이 날짜에 경험 남기기
이력서 후보 보기
성과 정보 추가하기
협업 맥락 보완하기
커리어 인사이트 보기
```

피해야 할 CTA:

```text
업데이트 추가
약한 항목 확인
리포트 보기
생성하기
```

## D-10. 완료 체크리스트

```text
[ ] 주요 섹션에서 `업무 관리`보다 `경험 정리` 톤이 우선 적용되었다.
[ ] `최근 리포트` 표현이 메인 섹션명에서 제거되었다.
[ ] `이력서 문장`보다 `이력서 후보` 표현이 우선 사용되었다.
[ ] `0건`, `기록 없음`, `없습니다`류 표현이 반복 강조되지 않는다.
[ ] 확정형/보장형 표현이 없다.
[ ] 부족 신호가 부정적 평가가 아니라 보완 방향으로 표현된다.
[ ] CTA에는 행동 이유 또는 기대 효과가 함께 있다.
[ ] 전체 화면 톤이 생산성 앱보다 커리어 분석 플랫폼에 가깝다.
```

## D-11. 작업 중 판단 규칙

1. 사용자에게 평가받는 느낌을 주지 않는다.
2. 가능성형 표현을 우선한다.
3. `리포트`보다 `인사이트`를 우선한다.
4. `이력서 문장`보다 `이력서 후보`를 우선한다.
5. 빈 상태는 부족함이 아니라 시작점으로 표현한다.
6. 과장/보장 표현은 쓰지 않는다.
7. 산출물 D 완료 체크리스트를 만족하지 못하면 완료로 보고하지 않는다.
