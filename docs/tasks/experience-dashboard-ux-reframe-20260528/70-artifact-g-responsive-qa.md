# 산출물 G 상세 정의: 모바일/반응형 기준

## G-0. 이 산출물의 목적

`HomeDashboard`의 경험 정리 대시보드 개편 결과가 데스크톱에서는 대시보드처럼 명확하게 보이고, 모바일에서는 정보가 깨지지 않고 자연스럽게 읽히도록 한다.

이번 작업은 데스크톱 화면의 정보 구조 개편이 중심이지만, 모바일에서 카드/캘린더/CTA가 깨지면 완료로 보지 않는다.

사용자는 모바일에서도 아래 흐름을 이해할 수 있어야 한다.

```text
최근 경험 분석 결과
→ 경험 흐름 캘린더
→ 최근 커리어 인사이트
→ 다음 추천 행동
```

---

## G-1. 전체 반응형 원칙

모든 영역은 아래 원칙을 따른다.

1. 모바일에서는 1열 구조를 기본으로 한다.
2. 데스크톱에서는 2열 또는 3열 grid를 허용한다.
3. 텍스트/칩/버튼이 카드 밖으로 넘치면 안 된다.
4. 가로 스크롤을 만들면 안 된다.
5. CTA는 모바일에서 세로로 쌓여도 자연스러워야 한다.
6. 정보 순서는 모바일에서도 `분석 결과 → 흐름 → 인사이트 → 행동` 순서를 유지한다.

---

## G-2. 상단 카드 반응형 기준

산출물 A의 상단 `최근 경험 분석 결과` 카드는 아래 기준을 따른다.

### 데스크톱

- 카드 내부를 2열로 나눌 수 있다.
- 좌측: 제목, 설명, 반복 경험 신호
- 우측: 연결 가능 직무, 상태 배지, CTA
- 카드가 너무 넓게 비어 보이지 않도록 좌우 영역 균형을 맞춘다.

권장 구조:

```text
lg:grid lg:grid-cols-[minmax(0,1.4fr)_minmax(280px,0.6fr)]
```

### 모바일

- 카드 내부는 1열이다.
- 제목 → 설명 → 경험 신호 → 연결 직무 → CTA 순서로 읽힌다.
- chip은 줄바꿈된다.
- CTA 2개는 세로 배치 또는 자연스러운 wrap을 허용한다.

피해야 할 것:

- 직무 chip이 한 줄에 억지로 들어가며 overflow 발생
- CTA 버튼이 카드 밖으로 넘어감
- 좌우 2열 구조가 모바일까지 유지됨

---

## G-3. 캘린더 반응형 기준

산출물 B의 `경험 흐름 캘린더`는 가장 깨지기 쉬운 영역이다.

### 데스크톱

권장 구조:

```text
캘린더 본문 좌측 / 선택 날짜 상세 우측
```

- 캘린더와 상세 패널을 2열로 배치 가능
- 선택 날짜 상세 패널은 캘린더보다 좁아도 된다.
- 날짜 셀은 균등한 grid를 유지한다.

### 모바일

모바일에서는 반드시 아래 순서를 따른다.

```text
경험 흐름 캘린더 헤더
월간 캘린더
선택 날짜 상세
다음 추천 행동
```

모바일 날짜 셀 기준:

- 날짜 숫자는 항상 보여야 한다.
- 경험 신호 텍스트가 길면 숨기거나 짧은 dot/chip만 표시한다.
- 긴 신호명은 선택 날짜 상세에서 보여준다.
- 날짜 셀 내부에 긴 문장을 넣지 않는다.

피해야 할 것:

- 캘린더가 화면 밖으로 가로 스크롤됨
- 날짜 셀 안에 긴 텍스트가 들어가 줄이 깨짐
- 선택 날짜 상세가 캘린더보다 먼저 나와 흐름이 어색함

---

## G-4. 하단 3단 흐름 반응형 기준

산출물 C의 하단 3개 카드 기준이다.

### 데스크톱

- 3열 grid 가능
- 카드 높이는 완전히 같지 않아도 되지만, 시각적 균형을 맞춘다.
- 카드 간 gap은 충분히 둔다.

권장:

```text
lg:grid-cols-3
```

또는 내용량 차이가 크면:

```text
lg:grid-cols-[1fr_1fr_1fr]
```

### 모바일

- 카드 3개는 1열로 쌓인다.
- 순서는 반드시 유지한다.

```text
1. 최근 경험 흐름
2. 현재 강점 및 부족 신호
3. 다음 추천 행동
```

- 각 카드 안 CTA는 card 하단에 위치한다.
- chip은 줄바꿈되어야 한다.

---

## G-5. CTA 반응형 기준

CTA는 모든 화면 폭에서 클릭 가능하고 읽기 쉬워야 한다.

### 데스크톱

- 2개 CTA를 가로로 배치 가능
- primary / secondary 우선순위가 명확해야 한다.

### 모바일

- CTA는 1열 또는 wrap 가능
- 버튼 너비는 `w-full` 또는 자연스러운 wrap을 사용한다.
- 버튼 텍스트가 줄바꿈되어도 어색하지 않아야 한다.

권장:

```text
flex flex-col gap-2 sm:flex-row
```

또는 카드 내부 CTA:

```text
w-full sm:w-auto
```

---

## G-6. 텍스트 길이 기준

모바일에서 긴 텍스트가 깨지지 않도록 아래 기준을 따른다.

- 카드 제목: 1줄 또는 자연스러운 2줄
- 설명 문구: 2~3줄 이내
- chip: 10~14자 내외 권장
- 기록 요약: 1~2줄 clamp 가능
- 추천 행동 설명: 2~3줄 내외

피해야 할 것:

- 한 카드 안에 긴 문단 3개 이상
- 모바일에서 읽기 어려운 dense text
- 한 줄에 너무 많은 chip

---

## G-7. 시각적 밀도 기준

개편 후 화면이 너무 복잡해지면 안 된다.

권장:

- 상단 카드: 가장 큰 시각적 우선순위
- 캘린더: 두 번째 우선순위
- 하단 3단 카드: 보조 인사이트

모바일에서는 spacing을 충분히 둔다.

권장 class 방향:

```text
space-y-6
space-y-8
p-4 sm:p-5 lg:p-6
gap-4 lg:gap-6
```

피해야 할 것:

- 모든 카드가 같은 강도로 강조됨
- 너무 많은 border/색상/배지가 한 화면에 몰림
- CTA가 너무 많아 선택이 어려움

---

## G-8. 접근성 기준

모든 interactive 요소는 아래 기준을 따른다.

- button 요소 사용
- focus style 유지
- hover style 유지
- aria-label 또는 title이 필요한 날짜 셀에는 의미 부여
- 색상만으로 상태를 전달하지 않음
- 상태 텍스트를 함께 제공

캘린더 날짜 셀은 특히 아래를 지킨다.

```text
선택 날짜
오늘
기록 있음
경험 신호 있음
```

위 상태는 색상 외에도 텍스트/title/aria-label 중 하나로 보조한다.

## G-10. 완료 체크리스트

```text
[ ] 모바일에서 상단 카드가 1열로 자연스럽게 보인다.
[ ] 모바일에서 캘린더가 가로 스크롤을 만들지 않는다.
[ ] 모바일에서 선택 날짜 상세가 캘린더 아래에 자연스럽게 배치된다.
[ ] 하단 3단 카드는 모바일에서 1열 순서로 쌓인다.
[ ] CTA는 모바일에서 카드 밖으로 넘치지 않는다.
[ ] chip/text가 overflow되지 않는다.
[ ] 데스크톱에서는 상단 카드/캘린더/하단 카드가 대시보드처럼 보인다.
[ ] 색상만으로 상태를 전달하지 않는다.
[ ] focus/hover 스타일이 유지된다.
```

## G-12. 작업 중 판단 규칙

1. 모바일은 1열 우선.
2. 데스크톱은 대시보드형 grid 허용.
3. 정보량보다 읽기 흐름을 우선.
4. 가로 스크롤은 만들지 않는다.
5. CTA는 항상 카드 안에 들어와야 한다.
6. 캘린더 날짜 셀에는 긴 문장을 넣지 않는다.
7. 산출물 G 체크리스트를 만족하지 못하면 완료로 보고하지 않는다.
