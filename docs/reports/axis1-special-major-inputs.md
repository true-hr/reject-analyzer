# Axis1 Special Major Bridge Inputs

**작성 목적**: 정보 부족, 조합 의존성, 포트폴리오 중심 등 특수한 전공 유형의 Registry 설계 입력 정보를 문서화.

**대상**: DOUBLE_MAJOR, CONVERGENCE_MAJOR, UNDECLARED_OTHER, OTHER_DESIGN

**검증 기준**: 
- 정보 부족/조합 의존성을 정직하게 표현
- 직무 연결을 과장하지 않음
- 사용자가 무엇을 추가로 제시해야 하는지 명확히 안내
- Round 1~3과 품질 대등성 유지

## 1. DOUBLE_MAJOR / 복수전공

**Canonical Key**: DOUBLE_MAJOR

**한글 전공명**: 복수전공

**대표 연결 직무**:
- 1순위: 두 전공 조합에 따라 결정 (예: 경제학+컴퓨터공학→데이터분석/PM, 심리학+경영학→HR/마케팅리서치)
- 2순위: 보조 역할 (한 전공 중심의 직무에서 다른 전공이 보조 역량)
- 보조 연결: 정보 부족 시 판단 불가

**연결 강도**: 정보부족/조합의존 - 두 전공 조합이 명확하고 교차점이 강할 때만 직접 연결

### majorDefinition

복수전공은 학위 요건을 만족하는 두 개의 전공을 병행 이수한 경우입니다. 강점은 "전공이 두 개"가 아니라 "두 전공을 어떻게 연결해 문제를 보고 풀 수 있는가"에 있습니다. 전공 조합에 따라 직무 연결 강도가 크게 달라집니다.

### jobConnection

복수전공의 직무 연결은 두 전공의 교차점에서 나옵니다.

**강한 연결 예**:
- 경제학 + 컴퓨터공학 → 데이터분석, 핀테크, PM (시장분석 + 기술 이해)
- 심리학 + 경영학 → HR, 마케팅리서치, UX리서치 (사용자 이해 + 조직/시장 맥락)
- 디자인 + 컴퓨터공학 → UI/UX, 제품개발 (시각/기능 설계 + 개발 이해)

**약한 연결 예**:
- 무관련 두 전공의 단순 나열 → 직접 연결 어려움
- 한 전공만 직무와 연결되는 경우 → "복수전공"이라는 장점 약화

따라서 실제 두 전공 조합을 알아야만 정교한 연결이 가능합니다.

### careerBridge

복수전공이 직무 역량으로 제시되려면 다음과 같은 구조가 필요합니다:

"저는 A전공에서 배운 [관점/방법론]과 B전공에서 배운 [관점/방법론]을 결합해, 희망 직무의 [구체 문제]를 볼 수 있습니다. 특히 [두 전공이 만나는 프로젝트/과목/경험]에서 이를 실제로 적용했습니다."

**예**:
- "경제학으로 시장을 읽고, 데이터분석 기술로 패턴을 추출해 제품 타깃을 결정하는 과정을 인턴에서 경험했습니다."
- "산업디자인의 형태 논리와 UX의 사용자 연구를 결합해, 신제품 기획 프로젝트에 참여했습니다."

다만 실제 두 전공 조합이 미상이면, 다음과 같이 표현:

"복수전공의 강점을 드러내려면 두 전공이 구체적으로 어떻게 만났는지, 특히 희망 직무와 가까운 프로젝트/과목/경험이 무엇인지 설명이 필요합니다."

### appealingCourses

특정 과목명보다 유형을 중심으로 작성:

- 주전공의 핵심 과목 (1순위 직무 기초)
- 복수전공의 핵심 과목 (교차점)
- 두 전공이 만나는 프로젝트 과목 (캡스톤, 졸업작품, 팀프로젝트)
- 교차 분야 과목 (데이터, 리서치, 기획, 설계)

**예**:
- "경제학 + 컴퓨터공학"
  - 경제학: 미시경제학, 산업조직론, 계량경제학
  - 컴퓨터공학: 데이터베이스, 머신러닝, 통계
  - 교차: 데이터 프로젝트, 경제 시뮬레이션

### evidencePrompts

- 두 전공을 함께 활용한 프로젝트/과목/과제가 있는가? (구체적으로 어느 부분에서 두 전공이 만났는가?)
- 한 전공의 문제를 다른 전공의 방법론으로 풀어본 경험이 있는가?
- 희망 직무와 가장 가까운 쪽의 전공이 무엇인가? 그 전공의 고급 과목/프로젝트는?
- 두 전공을 선택한 이유를 직무 관심과 연결해 설명할 수 있는가?
- 두 전공 중 약한 쪽 전공을 보충할 수 있는 자격/인턴/개인 프로젝트가 있는가?
- 실제 교차점이 약하다면, 추가 학습이나 프로젝트로 보완한 증거가 있는가?

### 과장 금지 표현

- "복수전공이라 융합형 인재입니다"
- "두 전공을 했으니 더 강합니다"
- "폭넓은 시각을 가지고 있습니다"
- "다양한 분야를 배웠습니다"
- "복수전공만으로 차별화됩니다"
- "모든 직무에 유리합니다"
- "전공 조합 자체가 강점입니다"

### QA 대표 케이스

- Major: DOUBLE_MAJOR
- TargetJob (구현 전 확인 필요): 전공 조합별로 다름 (BUSINESS_STRATEGY, DATA_ANALYSIS, PRODUCT_MARKETING_PMM, UI_UX_DESIGN 등 후보)
- ExpectedContains:
  - 두 전공의 구체적 이름 또는 카테고리 (예: "경제학과 컴퓨터공학")
  - 두 전공이 만나는 구체적 프로젝트/과목/경험
  - 교차점에서의 문제 해결 논리
- ShouldNotContain:
  - "융합형", "폭넓은", "다양한" 같은 추상 표현
  - "모든 직무", "어디든"
  - 전공 조합 정보 없는 일반론

### 구현 우선순위

**P2** — 정보 부족 케이스 (두 전공 조합을 알아야만 정교한 연결 가능)

## 2. CONVERGENCE_MAJOR / 융합전공

**Canonical Key**: CONVERGENCE_MAJOR

**한글 전공명**: 융합전공

**대표 연결 직무**:
- 1순위: 융합 구성 요소에 따라 결정 (예: 데이터+경영 → 비즈니스애널리틱스, 디자인+기술 → UI/UX/제품)
- 2순위: 프로젝트 기반 경험이 있을 때 강화
- 보조 연결: 커리큘럼/산출물 정보 부족 시 판단 불가

**연결 강도**: 정보부족/구성의존 - 융합의 구체적 구성과 프로젝트 산출물이 명확할 때만 강함

### majorDefinition

융합전공은 여러 학문 분야를 의도적으로 결합한 학위 프로그램입니다. "융합"이라는 이름은 매력적이지만, 실제 강점은 "무엇과 무엇을 어떻게 결합했는가"와 "구체적 산출물이 무엇인가"에 있습니다. 커리큘럼의 구성 요소, 프로젝트 경험, 손으로 만든 산출물(포트폴리오, 시뮬레이션, 설계 결과물 등)이 직무 적합성을 판단합니다.

### jobConnection

융합전공의 직무 연결은 "융합"이 아니라 구성 요소와 산출물로 판단됩니다.

**강한 연결 예**:
- 데이터과학 + 비즈니스 (경영) → 비즈니스애널리틱스, 마케팅데이터분석 (데이터 도구 + 비즈니스 문제 해석)
- 디자인 + 컴퓨터공학 → UI/UX/제품개발 (시각 설계 + 프로토타입/코드 이해)
- 콘텐츠 제작 + 미디어기술 → 콘텐츠마케팅, 영상제작 (제작 노하우 + 기술 도구)

**약한 연결 예**:
- "융합"이라는 이름만 있고 프로젝트 산출물이 명확하지 않은 경우
- 커리큘럼 구성 요소가 서로 거리가 먼 경우

따라서 실제 커리큘럼, 프로젝트 산출물, 손으로 만든 결과물을 알아야 정교한 연결이 가능합니다.

### careerBridge

융합전공이 직무 역량으로 제시되려면 다음과 같은 구조가 필요합니다:

"우리 전공은 [구성요소1]과 [구성요소2]를 결합합니다. 특히 [프로젝트/캡스톤 이름]에서 저는 [도구/방법론]을 사용해 [구체적 산출물/결과]를 만들었습니다."

**예**:
- "데이터과학 + 비즈니스 융합전공에서 인스타그램 마케팅 데이터를 분석해 타깃 세그먼트와 최적 게시 시간을 도출했습니다."
- "디자인 + 기술 융합전공에서 모바일 앱 프로토타입을 설계하고, Figma/Xcode로 구현했습니다."

다만 커리큘럼/산출물이 미상이면, 다음과 같이 표현:

"융합전공의 강점을 드러내려면 구체적 구성 요소(무엇과 무엇을 배웠는가), 주요 프로젝트, 손으로 만든 산출물(포트폴리오, 코드, 데이터 분석 결과 등)이 필요합니다."

### appealingCourses

구성 요소 중심으로 작성:

- 구성요소 1의 핵심 과목
- 구성요소 2의 핵심 과목
- 두 구성이 만나는 프로젝트 과목 (캡스톤, 팀프로젝트, 실무 프로젝트)
- 도구/기술 과목 (데이터분석 도구, 설계 도구, 개발 환경)

**예**:
- "데이터과학 + 비즈니스"
  - 데이터과학: 통계, 머신러닝, 빅데이터
  - 비즈니스: 마케팅, 경영전략, 고객분석
  - 통합: 데이터 프로젝트, 비즈니스케이스 분석

### evidencePrompts

- 융합전공의 두 (또는 그 이상의) 구성 요소가 무엇인가?
- 각 구성 요소의 핵심 과목은 무엇이고, 어떤 도구/스킬을 배웠는가?
- 캡스톤/졸업작품에서 구체적으로 무엇을 만들었는가? (포트폴리오, 코드, 데이터 분석 보고서 등)
- 팀프로젝트에서 희망 직무와 관련된 산출물이 있는가?
- 구성 요소 간의 교차점에서 실제로 문제를 푼 경험이 있는가?
- 희망 직무에 가장 가까운 프로젝트는 무엇이고, 그 결과물은 무엇인가?

### 과장 금지 표현

- "융합전공이라 미래형 인재입니다"
- "모든 신산업에 적합합니다"
- "다양한 분야를 배웠습니다"
- "융합 역량이 있습니다"
- "직무 적응력이 높습니다"
- "창의성과 기술을 모두 갖춤"
- "우리 전공은 특수합니다"

### QA 대표 케이스

- Major: CONVERGENCE_MAJOR
- TargetJob (구현 전 확인 필요): 구성요소에 따라 다름 (DATA_ANALYSIS, UI_UX_DESIGN, BUSINESS_STRATEGY, CONTENT_MARKETING 등 후보)
- ExpectedContains:
  - 구성 요소 명시 (예: "데이터과학과 비즈니스")
  - 주요 프로젝트 또는 산출물
  - 도구/기술 또는 결과물
- ShouldNotContain:
  - "융합형", "미래형", "특수한"
  - "모든 분야", "어디든"
  - 구성 요소 정보 없는 일반론
  - "창의성" 같은 추상 표현

### 구현 우선순위

**P2** — 정보 부족 케이스 (커리큘럼/산출물 정보 필요)

## 3. UNDECLARED_OTHER / 미정·기타

**Canonical Key**: UNDECLARED_OTHER

**한글 전공명**: 미정·기타

**대표 연결 직무**:
- 1순위: 판단 불가 (정보 부족)
- 2순위: 추가 정보 제공 후 재평가
- 보조 연결: 없음

**연결 강도**: 정보부족 - 추가 입력 정보(세부전공, 과목, 프로젝트, 자격, 인턴)가 있을 때만 판단 가능

### majorDefinition

"미정·기타"는 정보 부족 상태입니다. 전공 선택이 아직 이루어지지 않았거나, 기존 분류에 맞지 않는 독특한 전공 조합이거나, 설문 당시 전공명을 명시하지 않은 경우입니다. 이 경우 전공 자체의 내용이 아니라, 추가로 제공되는 정보(수강 과목, 프로젝트, 경험, 자격, 관심 산업)를 통해 직무 연결성을 보완할 수 있습니다.

### jobConnection

전공 정보가 명확하지 않으므로, Axis1만으로는 전공 기반 직무 연결을 강하게 판단하기 어렵습니다. 대신 다음과 같은 보충 정보가 있으면 해석 가능성이 높아집니다.

**보충 정보 유형**:
- 수강 과목 (어떤 과목들을 배웠는가?)
- 프로젝트 경험 (팀프로젝트, 캡스톤, 대회 참가)
- 인턴 경험 (어느 산업/직무 경험)
- 자격 (관련 자격증, 수료 증명)
- 도구/기술 (어떤 도구를 배웠는가?)
- 관심 산업/직무 (명시적 선호)

이 정보들을 조합하면, "전공명은 모르지만, 이런 경험과 관심이 있으니 이 직무에 관심 있음"으로 해석할 수 있습니다.

### careerBridge

"현재 Axis1(전공 정보)만으로는 직무와의 직접 연결을 판단하기 어렵습니다. 다음 정보가 있으면 더 정교한 분석이 가능합니다:

- 수강한 주요 과목/프로젝트 (문제 해결 방식을 이해하는 단서)
- 인턴/경험 (실제 산업/직무 접촉)
- 희망 산업/직무 (명시적 관심)
- 자격/기술 (도메인 역량 증명)

이런 정보가 함께 있으면, Axis3(경험 연결성)과 Axis5(역량 판단)에서 보충 분석이 가능합니다."

### appealingCourses

특정할 수 없음. 대신 사용자에게 "어떤 유형의 과목을 배웠는가"를 질문하는 형태로 유도:

- 데이터/분석 과목 (통계, 데이터베이스, SQL 등)
- 기술/개발 과목 (프로그래밍, 웹개발, 앱개발 등)
- 기획/전략 과목 (마케팅, 경영전략, 기획 등)

## 4. OTHER_DESIGN / 기타 디자인

**Canonical Key**: OTHER_DESIGN

**한글 전공명**: 기타 디자인

**대표 연결 직무**:
- 1순위: 세부 디자인 분야에 따라 다름 (제품디자인→제품개발, 산업디자인→PM/제품, 공간디자인→건축PM, 패션→브랜드)
- 2순위: 포트폴리오가 있을 때 강화
- 보조 연결: UI_UX_DESIGN 또는 BRAND_DESIGN

**연결 강도**: 포트폴리오의존 - 실제 산출물 포트폴리오가 있을 때만 강함

### majorDefinition

"기타 디자인"은 시각디자인이나 UI/UX가 아닌 다른 디자인 분야입니다. 제품디자인, 산업디자인, 공간디자인, 패션디자인, 서비스디자인, 공예, 커뮤니케이션디자인 등 다양한 세부 분야가 있습니다. 각 분야는 문제 정의, 사용자/고객 이해, 형태 또는 공간 설계, 프로토타입, 산출물 제작의 프로세스는 공통이지만, 직무 연결은 세부 분야에 따라 크게 달라집니다. 포트폴리오는 필수입니다.

### jobConnection

기타 디자인의 직무 연결은 세부 분야와 포트폴리오에 따라 달라집니다.

**강한 연결 예**:
- 제품디자인 → 제품개발, PM, 산업 엔지니어링 (사용자 중심 문제 정의, 프로토타입 제작)
- 산업디자인 → 제조 PM, 품질관리, 서비스 기획 (기능성/미학/생산성 통합)
- 공간디자인 → 건축 PM, 시설 관리, 공간 기획 (공간 제약/효율성/사용자 동선)
- 패션디자인 → 브랜드마케팅, 제품기획 (트렌드 분석, 소비자 심리, 브랜드 가치)
- 서비스디자인 → PM, 마케팅, HR (고객 경험 최적화, 프로세스 설계)

**약한 연결 예**:
- 세부 분야 미특정 + 포트폴리오 없음 → 판단 불가
- "디자인 감각" 중심 설명 + 구체 산출물 없음

### careerBridge

기타 디자인이 직무 역량으로 제시되려면 다음과 같은 구조가 필요합니다:

"저는 [세부 디자인 분야]을 공부했으며, [구체적 산출물/프로젝트]에서 [문제 해석/해결 과정]을 경험했습니다. 특히 [사용자/고객 니즈 분석, 형태/공간 설계, 프로토타입 제작, 시장 피드백]의 순환 과정을 이해하고 있습니다."

**예**:
- "제품디자인 전공으로 사용자 중심 문제 정의와 프로토타입 제작을 배웠으며, 캡스톤에서 휠체어 악세서리를 설계하고 3D 프린팅으로 제작한 포트폴리오가 있습니다."
- "공간디자인으로 동선/기능/미학을 통합 설계하는 훈련을 받았고, 카페/오피스/전시 공간 설계 프로젝트를 완료했습니다."

다만 세부 분야나 포트폴리오가 미상이면, 다음과 같이 표현:

"기타 디자인의 강점을 드러내려면 구체적 세부 분야(제품/산업/공간/패션/서비스 중 어느 것), 실제 포트폴리오(완성된 산출물, 프로토타입 또는 설계도), 그리고 그것이 해결한 구체적 문제가 필요합니다."

### appealingCourses

세부 분야별로 달라짐. 구분해서 제시:

**공통**:
- 디자인 사고 및 프로세스
- 사용자 조사 및 문제 정의
- 프로토타입 및 모델링
- 포트폴리오 제작

**제품/산업디자인**:

- 재료 및 제조 기술
- 3D 모델링 및 렌더링
- 제품 기획 및 마케팅

**공간디자인**:

**서비스디자인**:

### evidencePrompts
