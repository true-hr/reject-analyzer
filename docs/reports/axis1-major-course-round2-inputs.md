## 1. SOFTWARE / 소프트웨어

**Canonical Key**: SOFTWARE
**한글 전공명**: 소프트웨어
**대표 연결 직무**: 백엔드 개발, 웹 개발, 소프트웨어 엔지니어
**연결 강도**: 직접 연결

### majorDefinition
소프트웨어는 데이터를 받아 처리하고, 그 결과를 저장하며, 사용자와 상호작용하는 논리 구조를 설계하고 구현하는 전공입니다.

### jobConnection
백엔드 개발과 웹 개발의 핵심은 "데이터의 입력 → 처리 → 저장 → 출력"의 체인을 안정적이고 빠르게 구현하는 것입니다. 프로그래밍, 자료구조, 알고리즘은 이 처리 로직의 기초이고, 데이터베이스와 네트워크는 데이터 보관과 전달의 기초이며, 운영체제는 시스템 성능과 안정성의 기초입니다. 이 모든 요소를 함께 이해하는 것이 백엔드/웹 개발 직무의 본질입니다.

### careerBridge
따라서 소프트웨어 전공에서 배운 논리 설계, 데이터 구조 최적화, 에러 처리, 성능 개선의 사고방식이 직접 백엔드/웹 개발의 API 설계, 데이터 처리, 서버 구조 개선, 확장성 판단으로 연결됩니다.

### appealingCourses
- 프로그래밍 (C/C++, Java, Python, Go 등)
- 자료구조와 알고리즘
- 데이터베이스 (SQL, 정규화, 인덱싱)
- 운영체제 (프로세스, 메모리, I/O)
- 컴퓨터 네트워크 (TCP/IP, HTTP, DNS)
- 소프트웨어 공학 (설계, 테스트, 배포)
- 웹 프로그래밍 (서버/클라이언트 아키텍처)

### evidencePrompts
- 실제 프로젝트에서 구현한 API, 데이터 처리 로직, 최적화 경험
- GitHub 리포지토리, 코드 리뷰, PR 이력
- 버그 분석, 성능 개선 사례
- 배포, 모니터링, 장애 대응 경험

### 과장 금지 표현
- "완벽한 개발자", "모든 기술을 알고 있다"
- "프로젝트를 혼자 완성했다" (팀 협업 맥락 없이)
- "고도의 최적화를 했다" (구체적 지표 없이)
- "모든 언어를 다룬다"

### QA 대표 케이스
Case: 경제학/경영학 전공이 백엔드 개발로 지원하는 경우
- 강점: 데이터 분석, 논리적 사고
- 약점: 프로그래밍 실무 경험 부재
- 필요: 실제 구현/배포 포트폴리오

### Round 2 구현 우선순위
**P0** — Round 1 경험자 다수, Registry 설계 모두 완료

## 2. ELECTRICAL_ELECTRONIC / 전기전자

**Canonical Key**: ELECTRICAL_ELECTRONIC
**한글 전공명**: 전기전자
**대표 연결 직무**: 반도체 설계, 임베디드 개발, 하드웨어 엔지니어, 회로 설계
**연결 강도**: 직접 연결

### majorDefinition
전기전자는 전자기 신호와 반도체를 이용해 물리 세계의 데이터를 감지하고, 처리하고, 제어하는 장치를 설계·구현하는 전공입니다.

### jobConnection
반도체, 임베디드, 하드웨어의 핵심은 "센서 입력 → 신호 처리 → 제어 로직 → 액추에이터 출력"의 피드백 루프를 물리 레벨에서 구현하는 것입니다. 회로 설계, 신호 처리, 디지털/아날로그 제어는 이 루프의 각 단계에 필수적입니다. 안정성, 신뢰성, 전력 효율성은 제품의 생존 조건입니다.

### careerBridge
따라서 전기전자 전공에서 배운 회로 분석, 신호 해석, 제어 이론, 에러 조건 처리는 반도체 칩/시스템의 검증, 임베디드 펌웨어의 신뢰성 판단, 장애 원인 분석으로 직접 연결됩니다. 다만 제조 공정, 신뢰성 테스트, 양산 최적화 등 산업 경험이 함께 필요합니다.

### appealingCourses
- 회로 이론 (직류/교류, 임피던스)
- 디지털 논리 회로 및 설계
- 신호 및 시스템 (푸리에, 필터)
- 반도체 및 초대규모 집적회로 (VLSI)
- 마이크로컨트롤러 및 임베디드 시스템
- 제어 공학 (피드백, 안정성)
- 통신 공학 (변조, 복조)

### evidencePrompts
- 회로 설계 프로젝트, 시뮬레이션 (Spice, Cadence, 머리판)
- 임베디드 펌웨어 개발 (Arduino, STM32, RTOS)
- 신호 처리 구현, 필터 설계 검증
- 하드웨어 검증, 테스트 보고서
- 공정/회로 문제 분석, 개선 사례

### 과장 금지 표현
- "첨단 반도체를 설계했다" (실제는 학습 수준)
- "완벽한 제어 로직"
- "모든 신호 처리를 이해한다"
- "양산 경험이 있다" (학생 수준 프로젝트만 경험한 경우)

### QA 대표 케이스
Case: 전기전자 전공이 반도체 설계로 지원하는 경우
- 강점: 회로, 신호, 제어 이론
- 약점: 실제 칩/양산 경험 부재, 신뢰성 검증 미경험
- 필요: VLSI 프로젝트, 회로 검증 경험

### Round 2 구현 우선순위
**P0** — 산업 및 전공 수요 높음

## 3. CHEMISTRY / 화학

**Canonical Key**: CHEMISTRY
**한글 전공명**: 화학
**대표 연결 직무**: 품질관리, 생산기술, 제품 R&D 보조, 분석 시험
**연결 강도**: 직접 ~ 중간

### majorDefinition
화학은 물질의 구조와 반응을 이해하고, 원하는 성질의 물질을 만들기 위해 조건을 제어하는 것을 다루는 전공입니다.

### jobConnection
제조업의 품질관리, 생산기술, 분석 시험은 "원료 특성 파악 → 공정 조건 설정 → 산물 특성 검증 → 불량 원인 분석"의 사이클을 반복합니다. 화학 전공의 원리 이해와 실험 데이터 해석이 이 사이클의 기초입니다. 다만 특정 산업(반도체, 화학, 제약, 식음료, 바이오 등)의 제품과 공정을 깊이 있게 이해해야 실무에 가까워집니다.

### careerBridge
따라서 화학 전공에서 배운 분석 방법, 실험 설계, 데이터 해석은 품질 기준 검증, 공정 조건 이상 진단, 부작용/부산물 원인 규명으로 연결될 수 있습니다. 다만 특정 산업의 제품 특성, 공정 장비, 규제 기준을 함께 이해해야 합니다.

### appealingCourses
- 일반화학 및 화학실험
- 유기화학 및 실험
- 물리화학 (열역학, 반응 속도)
- 분석화학 (정량, 기기 분석)
- 화학공학 기초 (물질 수지, 에너지 수지)
- 고분자 화학
- 화학 안전 및 보건

### evidencePrompts
- 실험 설계, 데이터 분석 (그래프, 오차 분석)
- 분석 기기 활용 (HPLC, GC-MS, NMR, XRD 등)
- 부작용/이상 현상의 원인 추적
- 공정 조건과 산물 특성의 관계 파악
- 문헌 조사, 선행 사례 분석

### 과장 금지 표현
- "신약을 개발했다" (학부 수준 연구만 경험)
- "완벽한 공정 최적화"
- "모든 화학 지식을 알고 있다"
- "산업 경험을 충분히 했다" (학교 실험만 경험한 경우)

### QA 대표 케이스
Case: 화학 전공이 품질관리로 지원하는 경우
- 강점: 분석 방법, 실험 데이터 해석
- 약점: 산업별 공정/제품 특성 미숙, 대량 생산 경험 부재
- 필요: 인턴/프로젝트를 통한 산업별 제품/공정 이해

### Round 2 구현 우선순위
**P1** — 반도체/화학/제약 등 산업별로 세분화 필요

## 4. ACCOUNTING_TAX / 회계·세무

**Canonical Key**: ACCOUNTING_TAX
**한글 전공명**: 회계·세무
**대표 연결 직무**: 회계, 재무, FP&A, 세무
**연결 강도**: 직접 연결

### majorDefinition
회계는 조직의 거래를 일관된 규칙으로 기록하고, 그 결과를 재무제표로 요약하여, 재무 상황과 경영 성과를 해석 가능하게 만드는 전공입니다.

### jobConnection
회계 직무와 재무 직무의 핵심은 "거래 발생 → 기록 → 분류 → 보고 → 의사결정"의 정보 사이클입니다. 기록이 정확하지 않으면 모든 의사결정이 무너집니다. 거래를 정확히 기록하고, 재무제표를 읽고, 비용/수익의 흐름을 추적하는 것은 회계 전공의 핵심이며, 회계 직무의 필수 조건입니다.

### careerBridge
따라서 회계·세무 전공에서 배운 거래 기록, 계정 체계, 재무제표 작성, 세법 규정은 회계 처리, 예산 대비 실적 분석, 부문별 원가 관리, 재무 의사결정 지원으로 직접 연결됩니다.

### appealingCourses
- 회계학 원론
- 재무회계 (GAAP, IFRS)
- 관리회계 (원가, 예산, 성과 분석)
- 세법 (소득세, 법인세, 부가가치세)
- 재무제표분석
- 감사학
- 국제회계기준

### evidencePrompts
- 실제 거래 기록 및 재무제표 작성 경험
- Excel을 활용한 예산 대비 실적 분석
- 비용 구조 분석, 손익 추적 경험
- 세무 신고, 세무 이슈 분석 경험
- 재무 데이터 기반 경영진 보고서 작성

### 과장 금지 표현
- "완벽하게 회계를 할 수 있다" (실무 경험 없이)
- "모든 세법을 알고 있다"
- "대기업 회계 경험이 있다" (인턴만 경험한 경우)
- "세무 회피 전략을 만들었다"

### QA 대표 케이스
Case: 회계·세무 전공이 FP&A로 지원하는 경우
- 강점: 재무제표 이해, 비용/수익 분석
- 약점: 사업 이해, 데이터 분석, 커뮤니케이션 부족
- 필요: 부문별 재무 분석, 경영진 프레젠테이션 경험

### Round 2 구현 우선순위
**P0** — 경험자 다수, 개념 명확

## 5. BIO_LIFE_SCIENCE / 바이오·생명과학

**Canonical Key**: BIO_LIFE_SCIENCE
**한글 전공명**: 바이오·생명과학
**대표 연결 직무**: 제약 RA/MA, 바이오 품질관리, 영업기획, 임상시험 보조
**연결 강도**: 중간

### majorDefinition
바이오·생명과학은 세포, 유전자, 단백질, 생리 시스템을 이해하고, 그 원리를 이용해 질병을 진단하고 치료하는 기초를 다루는 전공입니다.

### jobConnection
제약·바이오 산업의 비임상/임상 역할은 "과학 근거 수집 → 안전성/효과성 판단 → 규제 기관 설득 → 현장 적용"의 과정입니다. RA(Regulatory Affairs), MA(Medical Affairs), 품질, 영업기획은 각 단계에서 "과학이 말하는 것"을 정확히 이해하고 "비과학적 주장"을 거르는 것이 중요합니다. 바이오 배경이 있으면 이 판단이 훨씬 빨라집니다.

### careerBridge
따라서 바이오·생명과학 전공에서 배운 약물 작용 원리, 질병 메커니즘, 임상 데이터 해석은 의약품 효능/부작용 이해, 규제 자료 작성, 고객(의료진/환자) 설명 자료 작성으로 연결될 수 있습니다. 다만 규제 환경, 임상 시험 프로세스, 사업 관점은 별도 경험으로 보강이 필요합니다.

### appealingCourses
- 세포생물학
- 유전학 (DNA, 유전자 발현)
- 생화학 (대사, 신호전달)
- 약리학 (약물 작용, 부작용)
- 병리학 (질병 메커니즘)
- 미생물학
- 분자생물학

### evidencePrompts
- 문헌 조사, 선행 연구 분석
- 실험 데이터 해석 (세포, 동물 모델)
- 약물 작용 메커니즘 설명 및 근거 제시
- 임상 데이터 해석
- 의료진/환자 대상 설명 경험

### 과장 금지 표현
- "신약을 개발했다" (학부/대학원 연구 수준만 경험)
- "완벽한 약리학 이해"
- "질병의 근본 원인을 알아낸다"
- "임상 경험이 충분하다" (학습 수준만 경험한 경우)

### QA 대표 케이스
Case: 바이오·생명과학 전공이 제약 RA로 지원하는 경우
- 강점: 약물 작용, 임상 이해
- 약점: 규제 환경, 문서화, 커뮤니케이션 미숙
- 필요: 규제 인턴/문서 작성 경험, 영어 능력

### Round 2 구현 우선순위
**P1** — 제약/바이오 기업별 편차 큼, 세분화 검토 필요

## 6. PHARMACY / 약학

**Canonical Key**: PHARMACY
**한글 전공명**: 약학
**대표 연결 직무**: 제약 RA/MA, 약국/의약품 품질관리, PM, 임상시험 코디네이터
**연결 강도**: 직접 ~ 중간

### majorDefinition
약학은 약물의 구조, 제형, 안전성, 유효성을 과학적으로 이해하고, 환자에게 올바르게 전달되도록 보장하는 전공입니다.

### jobConnection
제약 산업의 RA/MA/품질 역할은 "의약품의 과학적 근거를 규제 기관과 의료진과 환자에게 전달하는" 것입니다. 약학 전공은 약물의 약학적 성질(제형, 안정성, 생체이용률), 임상적 근거(효능, 안전성 프로필), 사용 조건(용법용량, 부작용)을 동시에 이해합니다. 이것이 제약 RA/MA의 가장 큰 강점입니다.

### careerBridge
따라서 약학에서 배운 의약품 구조 인식, 제제학, 약물학, 규제 과학은 의약품 허가 자료 작성, 의료진/환자 설명 자료 개발, 임상 이상반응 평가, 품질 기준 해석으로 직접 연결됩니다. 다만 사업 기획, 마케팅, 의료경제학 등은 별도 경험으로 보강이 필요합니다.

### appealingCourses
- 약학개론
- 약물학 (작용 기전, 부작용)
- 약제학 (제형, 제조, 안정성)
- 약동학/약력학
- 약사법 및 규제
- 임상약학
- 의약품 정보학

### evidencePrompts
- 의약품 허가 신청 자료 작성/검토 경험
- 임상 시험 데이터 분석
- 의료진 대상 학술 활동, 강의
- 약물 상호작용, 부작용 관리 경험
- 품질 기준 설정, 검증

### 과장 금지 표현
- "완벽한 의약품 이해" (실무 미경험)
- "신약 개발을 주도했다"
- "모든 약리학 지식이 있다"
- "임상 경험이 충분하다" (교육 수준만)

## 7. PUBLIC_POLICY / 행정·정책

**Canonical Key**: PUBLIC_POLICY
**한글 전공명**: 행정·정책
**대표 연결 직무**: 공공기관 기획, 사업기획, 정책기획, 행정
**연결 강도**: 직접 ~ 중간

### majorDefinition
행정·정책은 사회의 문제를 제도와 예산, 조직과 과정을 통해 해결하는 방법을 설계하고, 그 효과를 측정하는 것을 다루는 전공입니다.

### jobConnection
공공기관과 정책기획 직무의 핵심은 "사회 문제 정의 → 정책/사업 설계 → 예산 편성 → 이해관계자 조율 → 성과 측정"입니다. 행정·정책 전공은 이 전체 사이클을 이론적으로 이해하고, 실제 정책 사례를 분석합니다. 여기에 재무, 통계, 법률, 사회학을 함께 다루므로, 종합적 문제 해결 능력이 기본입니다.

### careerBridge
따라서 행정·정책에서 배운 정책 분석, 예산 편성, 이해관계자 분석, 성과 지표 설계는 공공 사업 기획, 정책 영향 평가, 행정 프로세스 개선으로 직접 연결될 수 있습니다. 다만 민간 사업 기획으로 옮기는 경우 시장 수요, 고객 관점, 수익성 판단을 새로 학습해야 합니다.

### appealingCourses
- 정책학개론
- 행정학개론
- 예산론 및 재정학
- 정책분석방법론
- 조사방법론
- 행정법
- 지역개발론
- 사회정책론

### evidencePrompts
- 정책 분석 논문, 사업계획서 작성
- 예산 편성, 성과 지표 설계 경험
- 이해관계자 인터뷰, 설문 조사 실시
- 정책 평가 보고서 작성
- 공공 프로젝트의 문제점 분석 및 개선안 제시

### 과장 금지 표현
- "정책을 만들었다" (학부 과제 수준)
- "완벽한 정책 분석"
- "모든 행정 지식이 있다"
- "정부와 협력한 경험" (강의나 초대 수준만)

### QA 대표 케이스
Case: 행정·정책 전공이 민간 사업기획으로 지원하는 경우
- 강점: 문제 정의, 이해관계자 분석, 성과 지표 설계
- 약점: 시장 수요 이해, 고객 중심 사고, 수익성 분석 미흡
- 필요: 시장 조사, 고객 인터뷰, 비즈니스 모델 분석 경험

## 8. SOCIOLOGY / 사회학

**Canonical Key**: SOCIOLOGY
**한글 전공명**: 사회학
**대표 연결 직무**: HR, 사용자 리서치, 마케팅 리서치, 조직문화
**연결 강도**: 중간

### majorDefinition
사회학은 인간이 집단 속에서 어떻게 행동하고, 어떤 문화와 제도에 영향받으며, 어떻게 변화하는지를 과학적으로 분석하는 전공입니다.

### jobConnection
HR, 조직문화, 사용자 리서치의 핵심은 "개인의 행동과 태도가 집단 맥락과 제도에 어떻게 영향받는가"를 이해하는 것입니다. 사회학 전공은 집단 행동, 문화, 제도, 불평등, 변화를 분석하는 렌즈를 제공합니다. 여기에 조사방법론, 통계를 함께 사용하면, 직원 설문, 고객 인터뷰, 조직 진단을 과학적으로 설계할 수 있습니다.

### careerBridge
따라서 사회학에서 배운 문화 분석, 집단 행동 해석, 조사 설계, 데이터 분석은 직원 태도 조사, 조직 문제 진단, 사용자 행동 연구, 마케팅 타겟 분석으로 연결될 수 있습니다. 다만 비즈니스 맥락(수익, 고객, 경쟁)과 조직 내 정치를 함께 이해해야 합니다.

### appealingCourses
- 사회학 개론
- 사회통계학 (기술, 추론)
- 조사방법론 (표본, 설문, 인터뷰)
- 조직사회학
- 산업사회학
- 문화사회학
- 사회계층론

### evidencePrompts
- 설문 조사 설계 및 분석 (표본 추출, 통계)
- 인터뷰 기획 및 분석 (코딩, 주제 추출)
- 조직/고객 데이터 해석 및 보고서 작성
- 문화 차이 분석, 집단 간 차이 도출
- 실행 가능한 개선안 제시 경험

### 과장 금지 표현
- "완벽한 조직 문화 이해" (개론 수준)
- "모든 사회 현상을 분석할 수 있다"
- "대규모 설문 조사를 주도했다" (도움만 주는 경우)
- "사회학 지식으로 모든 HR 문제를 풀 수 있다"

### QA 대표 케이스
Case: 사회학 전공이 사용자 리서치로 지원하는 경우
- 강점: 집단 행동 분석, 조사 설계, 통계
- 약점: 제품/서비스 이해, 데이터 도구 (분석 프로그래밍) 미흡
- 필요: 제품 조사 프로젝트, R/Python 분석 경험

### Round 2 구현 우선순위
**P1** — 리서치 역할별로 세분화 가능

## 9. OTHER_HUMANITIES / 기타 인문

**Canonical Key**: OTHER_HUMANITIES
**한글 전공명**: 기타 인문 (문학, 역사, 철학, 언어학 등)
**대표 연결 직무**: 콘텐츠 기획, 브랜드/마케팅, HR, 커뮤니케이션
**연결 강도**: 약함 ~ 중간

### majorDefinition
인문학은 인간의 사상, 문화, 언어, 역사를 깊이 있게 분석하고, 그 맥락 속에서 의미를 찾는 학문입니다.

### jobConnection
콘텐츠 기획, 브랜드, 커뮤니케이션의 핵심은 "대상 관객의 관심, 가치, 맥락을 이해하고, 그에 맞는 메시지를 구성하는" 것입니다. 인문학 전공은 텍스트 해석, 역사적 맥락 이해, 문화 분석을 훈련하므로, "왜 이 메시지가 이 관객에게 먹히는가"를 설명할 수 있습니다. 다만 비즈니스 성과(조회수, 전환율, 브랜드 인지도)와 연결하는 능력이 별도로 필요합니다.

### careerBridge
따라서 인문학에서 배운 텍스트 분석, 메시지 구성, 맥락 해석, 자료 요약은 콘텐츠 스크립트 작성, 브랜드 스토리텔링, 직원 커뮤니케이션, 고객 관점 설명으로 연결될 수 있습니다. 다만 구체적인 콘텐츠 제작(글쓰기, 영상, 디자인), 데이터 분석, 성과 측정 경험이 함께 필요합니다.

### appealingCourses
- 문학개론/문예창작
- 한국사/세계사
- 철학 (인식론, 윤리)
- 언어학/문법론
- 문화연구
- 수사학/설득 이론
- 기호학

### evidencePrompts
- 콘텐츠 제작 경험 (블로그, SNS, 뉴스레터)
- 브랜드 스토리, 캠페인 메시지 작성
- 직원/고객 인터뷰 및 인사이트 추출
- 자료 조사, 문맥 분석, 요약 및 재구성
- 글쓰기 성과 (조회, 반응, 공유)

### 과장 금지 표현
- "완벽한 글쓰기 능력" (구체적 포트폴리오 없이)
- "모든 문화를 이해한다"
- "콘텐츠 전략을 수립했다" (브레인스토밍만)
- "유명 기관과 협력했다" (일회성 강의만)

### QA 대표 케이스

- 강점: 텍스트 분석, 메시지 구성, 맥락 이해
- 약점: 제품/시장 이해, 데이터/성과 측정, 디자인 미흡
- 필요: 실제 브랜드 프로젝트, 포트폴리오, 마케팅 기초 학습

## 10. USER_EXPERIENCE / UX/HCI

**Canonical Key**: USER_EXPERIENCE
**한글 전공명**: UX/HCI
**대표 연결 직무**: UX 리서치, 서비스 기획, PM, 프로덕트 디자인
**연결 강도**: 직접 연결

### majorDefinition
UX/HCI는 사용자가 제품이나 서비스를 어떻게 이해하고, 상호작용하고, 만족하는지를 과학적으로 연구하고, 그 이해를 바탕으로 더 나은 경험을 설계하는 전공입니다.

### jobConnection
UX 리서치와 서비스 기획의 핵심은 "사용자 문제 파악 → 원인 분석 → 해결안 설계 → 검증"입니다. UX/HCI 전공은 이 전체 사이클을 과학적으로 수행하는 방법을 직접 훈련합니다. 사용자 조사, 사용성 테스트, 정보구조 설계, 프로토타입 제작은 UX/HCI 교육의 표준 과정입니다.

### careerBridge
따라서 UX/HCI에서 배운 사용자 조사 방법(인터뷰, 관찰, 태스크 분석), 사용성 테스트, 정보구조 설계, 프로토타입 평가는 사용자 문제 정의, 제품 흐름 설계, 개선 우선순위 판단, 디자인 검증으로 직접 연결됩니다.

### appealingCourses
- HCI 기초
- 사용자 조사 및 방법론
- 사용성 평가 및 테스트
- 정보 구조 설계 (IA)
- 인터랙션 디자인
- 프로토타이핑 및 설계 도구
- 접근성 및 포용적 설계

### evidencePrompts
- 사용자 인터뷰, 관찰, 태스크 분석 실행
- 페르소나, 사용자 여정 맵 작성
- 와이어프레임, 프로토타입 제작
- 사용성 테스트 진행 및 개선안 도출
- 포트폴리오 (프로젝트 사례, 근거, 결과)

### 과장 금지 표현
- "완벽한 UX 설계" (검증 없이)
- "모든 디자인 도구를 다룬다"
- "사용자 경험을 모두 이해한다"
- "유명 서비스를 설계했다" (팀 프로젝트만)

### QA 대표 케이스
Case: UX/HCI 전공이 PM/서비스기획으로 지원하는 경우
- 강점: 사용자 조사, 정보구조 설계, 사용성 검증

- 필요: 비즈니스 메트릭, 시장 이해, 기획/전략 프로젝트
