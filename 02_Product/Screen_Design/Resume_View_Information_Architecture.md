# Resume View Information Architecture

## 화면 목적
`이력서 보기`는 이력서 생성 결과 대시보드가 아니라, 실제 이력서 문서처럼 읽히는 화면이어야 한다.

- 이 화면의 목적은 분석 결과를 설명하는 것이 아니라 제출 가능한 이력서 초안의 형태를 보여주는 것이다.
- 사용자는 이 화면에서 문서 구조를 먼저 읽어야 하며, 내부 비교 로직이나 보조 설명을 먼저 읽어서는 안 된다.

## 이번 라운드 핵심 판단
- `이력서 보기`는 실제 이력서 문서 프리뷰다.
- `이력서 업데이트`는 기록 입력 / 수정 / 추가 / 반영 준비 화면이다.
- 결과 대시보드형 구조는 `이력서 보기`에서 축소 / 보조화 대상이다.

## 잘못 잡았던 방향과 교정
- 잘못 밀었던 방향
  - before / after 비교 중심
  - 강점 카드 중심
  - 최종 결과 카드 중심
- 이 방향은 `이력서 보기`가 아니라 `이력서 생성 결과 대시보드`에 가까웠다.
- 교정 방향
  - 문서 본문이 첫 화면 주인공이어야 한다.
  - 비교 / 연결 근거 / 설명 카드는 필요해도 보조 위치로 내려가야 한다.

## 이력서 보기 / 이력서 업데이트 역할 분리
### 이력서 보기
- 실제 이력서 문서 프리뷰
- 분석 화면 아님
- 비교 툴 아님
- 사용자가 위에서 아래로 문서처럼 읽을 수 있어야 한다

### 이력서 업데이트
- 오늘 기록 / 이번 주 기록 입력
- 수정 / 추가 / 반영 준비
- 비교 블록이 필요하면 이 화면에서 보조 기능으로 다룬다

## 화면 상단 군더더기 제거 원칙
- 상단 배지 / 설명 / 탭성 요소 중복 금지
- 본문이 빨리 시작되어야 한다
- 제거 / 축소 대상
  - `이력서 보기`
  - `문서형 프리뷰`
  - 긴 설명문
  - `이력서 결과`
  - `최근 기록 기반 이력서 초안`
  - `오늘 기록 기준`
  - 연결 근거의 전면 노출
  - 과도한 설명형 카드

## 화면에 남길 것
- 문서 제목
- 우측 액션 버튼
- 실제 이력서 본문
- 필요 시 매우 짧은 1줄 설명

## 화면에서 제거할 것
- 상단 반복 헤더성 배지
- 본문 시작 전에 여러 번 반복되는 역할 설명
- 결과 대시보드 느낌의 비교 블록 전면 배치
- 강점 카드 / 최종 결과 카드의 과도한 강조

## 액션 버튼 원칙
- 버튼은 내부 처리 상태가 아니라 사용자 기준 액션이어야 한다
- 확정 라벨
  - `이력서 가져오기`
  - `이력서 내보내기`
  - `이력서 다운로드`
- 위계 기준
  - 가져오기 = outline / secondary
  - 내보내기 = outline 또는 중간 강조
  - 다운로드 = primary 가능

## 이력서 문서 구조
- 이름 / 연락처 / 이메일 / 링크
- 소개
- 경력
- 주요 성과
- 학력
- 보유 역량
- 기타

## 최신 반영 상태

### 상단 군더더기 제거 반영
- 상단 반복 헤더성 요소 제거가 실제 코드에 반영되었다.
- 제거/비노출 완료 대상
  - `이력서 보기`
  - `문서형 프리뷰`
  - 긴 설명문
  - `이력서 결과`
  - `최근 기록 기반 이력서 초안`
  - `오늘 기록 기준`

### 현재 상단 구성의 최종 원칙
- 제목
- 1줄 설명
- 액션 버튼 3개
- 바로 본문 시작

`이력서 보기`는 상단 배지/설명/탭을 반복 노출하는 화면이 아니라,
제목과 최소 액션만 둔 뒤 실제 이력서 본문이 빠르게 시작되는 화면이어야 한다.

### 액션 버튼 라벨 최신 상태
- `이력서 가져오기`
- `이력서 내보내기`
- `이력서 다운로드`
- 버튼 배치는 작은 폭에서도 자연스럽게 줄바꿈되도록 `flex-wrap`을 유지한다.

### 아직 미정인 것
- 버튼 실제 기능 정의
- `내보내기`와 `다운로드`의 역할 차이
- placeholder 성격이 남아 있는 액션 처리 정책

### 다음 우선순위: 본문 문장 디테일
- 소개 문단 품질
- 경력 bullet 품질
- 성과 문장 품질

현재 `이력서 보기`의 다음 개선 우선순위는 정보구조가 아니라,
소개/경력/성과 문장의 실무적 설득력 보강이다.
