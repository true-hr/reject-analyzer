# Newgrad Test Baseline

## 문서 목적

- 패스맵 신입 Transition-Lite 엔진의 현재 수동 검증 상태를 한 페이지에서 빠르게 파악하기 위한 baseline 문서다.
- 이미 검증한 케이스, 현재까지의 판정, 남은 리스크, 다음 테스트 우선순위를 정리한다.
- 반복 테스트를 막기 위한 체크리스트를 함께 유지한다.

## 현재까지 검증 완료한 케이스

- A/B 기준선 테스트
- 프로젝트 보완형
- 인턴/단기경험 보완형
- self-report 과대상승 점검형

## 케이스별 판정 요약 표

| 케이스 | 핵심 목적 | 판정 | 핵심 결론 |
| --- | --- | --- | --- |
| A/B 기준선 테스트 | 상한/하한 분리 확인 | PASS | 상한/하한 분리는 대체로 정상이며 self-report는 axis5 중심으로만 작동 |
| 프로젝트 보완형 | 프로젝트가 전공 약점을 얼마나 메우는지 확인 | PASS | 프로젝트 2개가 axis1·axis3을 높음으로 보완, axis2는 낮게 유지 |
| 인턴/단기경험 보완형 | 인턴/단기경험이 axis2·axis4를 얼마나 보완하는지 확인 | 주의 | axis2는 3점, axis4는 5점까지 보완됐지만 axis1은 낮게 남음 |
| self-report 과대상승 점검형 | self-report가 다른 축까지 오염시키는지 확인 | PASS | axis5는 5점까지 상승, 다른 4축은 2/2/1/1로 낮게 유지 |

## 현재까지의 핵심 판단

### 상한/하한 분리 정상 여부

- 정상에 가깝다.
- A/B 기준선에서 상단과 하단 케이스가 대체로 분리됐다.

### 프로젝트 보완 작동 여부

- 작동한다.
- 프로젝트는 전공 약점을 남겨 둔 채 `직무 구조 연결성`, `역할 범위 연결성`을 실질적으로 끌어올린다.
- 반면 `산업 맥락 연결성`은 프로젝트만으로 크게 올라가지 않는다.

### 인턴/단기경험 보완 작동 여부

- 작동한다.
- 인턴/단기경험은 `산업 맥락 연결성`과 `고객 유형 연결성`에 실제 반응을 만들었다.
- 특히 axis4는 이해관계자/직접 소통 입력에 강하게 반응한다.

### self-report 오염 여부

- 제한적이다.
- self-report는 axis5를 강하게 끌어올리지만, 현재까지 확인된 범위에서는 다른 4축 점수까지 과도하게 밀어 올리지는 않았다.
- 다만 axis4 설명 surface에는 보조 신호가 일부 섞일 수 있어 지속 관찰이 필요하다.

## 현재 남은 리스크

- 설명 surface 누락
  - whyThisRead 상단에서 입력한 강점, workstyle, 단기 실무 일부가 충분히 드러나지 않는 경우가 있다.
  - 핵심 근거가 축 내부 설명에만 남고 상단 요약에서 빠지는 현상이 있다.
- axis2 산업 맥락 보수성
  - 프로젝트 보완형에서는 axis2가 낮게 남는다.
  - 인턴/단기경험 입력이 들어오면 3점까지는 오르지만, 상단 축으로 가는 조건은 여전히 보수적이다.

## 다음 우선 테스트

- 1순위: axis5 상단 설명 surface 점검
  - whyThisRead와 상단 헤더에서 self-report/경험 근거가 어떻게 요약되는지 확인
- 2순위: axis2 보수성 추가 검증
  - 산업 직접성은 약하지만 반복 노출이 있는 케이스에서 axis2가 어디까지 오르는지 확인
- 3순위: explanation surface와 selection pack 일치성 점검
  - 실제 선택된 근거가 표면 문구에 누락 없이 반영되는지 확인

## 반복 테스트 방지 체크리스트

- 이미 돌린 케이스를 baseline 확인 없이 다시 반복하지 않는다.
- 새 테스트 전 이 문서와 `05_Execution/Newgrad_TransitionLite_Manual_Test_Log.md`를 먼저 확인한다.
- PASS / 주의 / 이상 기준을 임의로 바꾸지 않는다.
- 축 점수와 설명 surface 문제를 혼동하지 않는다.
- 동일한 목적의 케이스를 다시 돌릴 때는 입력 변형 이유를 먼저 적는다.
